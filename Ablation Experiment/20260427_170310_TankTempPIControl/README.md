# TankTempPIControl 模型文档

## 1) 模型目的与范围

**模型目的**：基于 Modelica 标准库（MSL）的控制组件，实现保温水箱温度的闭环控制系统，模拟并验证水箱在环境温度阶跃扰动下的保温与自动调节能力。

**物理机制与范围**：
- **集中参数热容**：将水箱内的水视为单一集中热容节点。
- **热传导散热**：水箱向环境的热量损失与环境温差成正比。
- **连续 PI 控制**：采用比例-积分控制器调节加热器功率。
- **闭环反馈调节**：实时监测水温并反馈至控制器，与参考温度比较生成控制信号。

## 2) 组件清单

本模型使用了以下 Modelica 标准库（MSL）组件：

| 组件路径 | 实例名 | 功能描述 |
| :--- | :--- | :--- |
| `Modelica.Blocks.Sources.Constant` | `refTemp` | 产生恒定的参考温度设定值信号（K） |
| `Modelica.Blocks.Sources.Step` | `ambTemp` | 产生带阶跃扰动的环境温度信号（K） |
| `Modelica.Blocks.Continuous.LimPID` | `piController` | 带输出限幅与抗积分饱和的 PI 控制器 |
| `Modelica.Blocks.Math.Gain` | `heaterGain` | 将控制器输出信号转化为加热功率（W） |
| `Modelica.Blocks.Sources.RealExpression` | `waterTempExpr` | 将物理量水温转化为控制块图信号 |

| 变量名 | 值 | 描述 |
| :--- | :--- | :--- |
| `T_water` | 60.0 °C (333.15 K) | 水箱初始温度（严格锁定 `fixed=true`） |

*注：PI 控制器的内部状态及输出已根据稳态初始条件（`T_water = T_ref`）进行初始化，确保系统从稳态起步。*

## 5) 控制逻辑说明

本模型采用经典的闭环反馈控制逻辑：
1. **设定值与反馈**：`refTemp` 提供目标温度信号（转换为开尔文），`waterTempExpr` 捕获实际水温 `T_water` 作为反馈信号。
2. **误差计算**：PI 控制器 `piController` 的输入端接收设定值与反馈值，内部计算温度误差 `T_error = T_ref_signal - T_water`。
3. **PI 运算与限幅**：控制器根据比例增益 `Kp` 和积分时间 `Ti` 计算控制信号，并限制在 `[0, 20]` 范围内，同时具备抗积分饱和功能。
4. **功率转换**：控制器输出 `u_pi` 经过增益模块 `heaterGain` 放大 `k_heater` 倍，转化为实际加热功率 `P_heat`。
5. **扰动响应**：在 `t_disturb = 100s` 时，环境温度由 20°C 骤降至 10°C，散热量 `P_loss` 突增导致水温下降，控制逻辑自动提升 `P_heat` 以克服扰动，使水温重新跟踪设定值。

## 7) 结果概览

在 300 秒的仿真过程中，系统经历了初始稳态及 100 秒时的环境阶跃扰动，核心目标变量的统计摘要如下：

| 变量名 | 起始值 | 终值 | 最小值 | 最大值 | 单位 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `T_water` | 333.15 | 332.99 | 332.44 | 333.15 | K |
| `T_water_C` | 60.00 | 59.84 | 59.29 | 60.00 | °C |
| `T_ref_signal` | 333.15 | 333.15 | 333.15 | 333.15 | K |
| `T_error` | 0.00 | 0.16 | 0.00 | 0.71 | K |
| `u_pi` | 4.00 | 5.02 | 4.00 | 5.05 | 1 |

*分析：系统在 100s 环境温度下降后，最大温度误差约 0.71K，控制器迅速增加加热功率（最高至约 5052 W），使水温逐渐回升并趋近于设定值 60°C，最终稳态误差缩小至 0.16K。*

5. 配置仿真设置：停止时间为 `300`，求解器选择 `dassl`，容差设为 `1e-06`。
