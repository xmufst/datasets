# SpringEndStop 模型文档

## 1) 模型目的与范围

`SpringEndStop` 模型模拟了一个带末端挡块（接触碰撞）的质量-弹簧-阻尼器系统。模型的主要目的是演示使用 **罚函数法（Penalty Method）** 处理接触碰撞的动力学行为，并进行基本的能量分析。

**模型范围包括：**
*   实现经典的质量-弹簧-阻尼器动力学。
*   在质量块位移超过预设的间隙（`gap`）时，激活接触力模型。
*   接触力由接触弹簧（高刚度）和接触阻尼器组成，仅在穿透时产生。
*   计算并输出系统的主要动力学变量（位移、速度、力）和能量项（动能、弹簧势能、总机械能）。

## 2) 组件清单 (MSL 组件类型)

此模型未直接使用 Modelica 标准库 (MSL) 中的物理组件，而是通过方程实现了其物理行为。其功能等价于以下 MSL 组件的组合：

| 功能 | 等效 MSL 组件 (来自 `Modelica.Mechanics.Translational.Components`) |
| :--- | :--- |
| 质量 | `Mass` |
| 线性弹簧 | `Spring` (原点在 `x=0`) |
| 线性阻尼器 | `Damper` |
| 接触力 | 未直接对应，类似自定义力元，使用 `if` 语句和 `penalty` 方法实现。 |

## 3) 参数表

下表列出了模型的所有可调参数。

| 参数名 | 值 | 单位 | 描述 |
| :--- | :--- | :--- | :--- |
| `m` | 1.0 | kg | 质量块的质量 |
| `k` | 50.0 | N/m | 主弹簧的刚度 |
| `c` | 0.5 | N·s/m | 主阻尼器的阻尼系数 |
| `x0` | 0.8 | m | 初始位移参数（质量块的起始位置） |
| `v0` | 0.0 | m/s | 初始速度参数（质量块的起始速度） |
| `gap` | 0.5 | m | 到末端挡块的间隙距离 |
| `k_contact` | 1000.0 | N/m | 接触弹簧的刚度 |
| `c_contact` | 5.0 | N·s/m | 接触阻尼器的阻尼系数 |

## 4) 初始条件

模型的初始条件由参数 `x0` 和 `v0` 通过变量 `x` 和 `v` 的 `start` 属性设置。

| 变量 | 初始值 | 描述 |
| :--- | :--- | :--- |
| `x` | 0.8 m | 质量块的初始位移。此值大于 `gap`(0.5 m)，因此仿真开始时系统已处于接触状态。 |
| `v` | 0.0 m/s | 质量块的初始速度。 |

## 5) 控制逻辑说明

模型本身没有外部控制逻辑。其内部包含一个关键的**离散状态切换逻辑**，用于处理接触事件：

*   **接触判断**：连续变量 `penetration = x - gap` 用于计算质量块对末端挡块的穿透深度。
*   **状态切换**：
    *   当 `penetration > 0` 时，布尔变量 `in_contact` 和离散变量 `contact` 为 `true`，接触力 `F_contact` 被激活。
    *   当 `penetration <= 0` 时，`in_contact` 和 `contact` 为 `false`，接触力 `F_contact` 为零。
*   **作用**：离散变量 `contact` 的切换会触发 Modelica 求解器的事件处理机制，确保在接触边界（`x = gap`）进行精确的积分步长调整。这是一个基于物理量（位移）的开关逻辑。

## 6) 仿真设置

本模型的默认仿真设置已在模型代码的 `annotation` 中定义，并与输入要求一致。

*   **仿真时间**： `0.0` 至 `10.0` 秒。
*   **求解器**： `dassl` (默认的 DAE 求解器)。
*   **相对容差**： `1e-6`。
*   **输出**：建议设置固定步长输出间隔为 `0.01` 秒以便于分析，这通常由仿真环境的输出设置控制。

## 8) 运行与复现方法

1.  **加载模型**：
    确保模型文件 `./model/SpringEndStop.mo` 在您的 Modelica 环境（如 Dymola、OpenModelica）的工作路径下。

2.  **执行仿真**：
    使用以下实验设置进行仿真，即可复现文档所述结果：
    *   **停止时间**： `10.0`
    *   **求解器**： `dassl`
    *   **相对容差**： `1e-6`
    *   **输出间隔**： `0.01` (可在仿真设置中指定)

3.  **复现特定案例**：
