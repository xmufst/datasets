# BouncingBall Model Documentation

## 1) 模型目的与范围
本模型是一个弹跳球（BouncingBall）模型，用于模拟一个小球在重力作用下自由下落、与地面发生碰撞并反弹的物理过程。模型的核心目的是研究在考虑能量损失（非完全弹性碰撞）和数值稳定性（防止Zeno现象）的情况下，小球高度与速度随时间的变化规律。其范围仅限于垂直方向的运动，并假设地面为刚性平面。

## 2) 组件清单
此模型未直接使用Modelica标准库（MSL）中的预定义物理组件。它通过基础方程和事件处理逻辑（`when`语句）直接定义了系统的动力学行为和控制逻辑。因此，它是一个基于方程的简单物理系统模型。

## 3) 参数表
下表列出了模型的所有关键参数。

| 参数名 | 值 | 单位 | 描述 |
| :--- | :--- | :--- | :--- |
| `g` | 9.81 | m/s² | 重力加速度 |
| `e` | 0.7 | 1 | 恢复系数（碰撞后与碰撞前速度绝对值之比） |
| `h0` | 10.0 | m | 初始释放高度 |
| `v0` | 0.0 | m/s | 初始速度 |
| `v_threshold` | 0.01 | m/s | Zeno截断速度阈值（当碰撞速度低于此值时，强制设为零，使球停止弹跳） |

## 4) 初始条件
模型的连续状态变量初始值由参数定义，如下所示：
*   高度 `h`：初始值为 `h0` = 10.0 m。
*   速度 `v`：初始值为 `v0` = 0.0 m/s。

## 5) 控制逻辑说明
模型没有外部的控制逻辑，但内置了处理物理碰撞的离散事件逻辑，其作用类似于一个自动的物理响应“控制器”。
1.  **碰撞检测**：当小球高度 `h` 小于等于0（即接触地面）时，触发 `when` 事件。
2.  **速度重置**：在碰撞事件中，根据恢复系数 `e` 更新速度。若碰撞前速度 `pre(v)` 的绝对值大于 `v_threshold`，则速度反向并乘以系数 `e`（`v_new = -e * pre(v)`），模拟能量损失；否则，将速度置零，小球停止运动。这是防止“Zeno现象”（无限次碰撞在有限时间内发生）的关键机制。
3.  **位置重置**：碰撞后，将高度 `h` 重新初始化为0，确保小球从地面开始反弹。

## 6) 仿真设置
本次仿真的配置如下：
*   **终止时间**：10.0 s
*   **输出步长**：0.01 s
*   **求解器**：Dassl
*   **容差**：1.0e-6

## 7) 结果概览
对目标变量（`h`， `v`）的仿真结果统计摘要如下：

| 变量 | 单位 | 初始值 | 终值 | 最小值 | 最大值 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **`h` (高度)** | m | 10.0 | 0.0 | ≈-1.01e-10 | 10.0 |
| **`v` (速度)** | m/s | 0.0 | 0.0 | -14.007 | 9.805 |

**结果解读**：小球从10米高度释放，在重力作用下加速下落，与地面发生多次非完全弹性碰撞（`e=0.7`），每次反弹的高度和速度逐渐减小。仿真结束时，小球速度已低于设定的阈值（0.01 m/s），因此停止在地面上（`h=0， v=0`）。高度最小值出现的微小负值属于数值计算误差，在容差允许范围内。最大速度约14.0 m/s出现在某次碰撞前瞬间。

## 8) 运行与复现方法
1.  **模型文件**：本模型定义于 `./model/BouncingBall.mo`。
2.  **仿真环境**：需要一个支持Modelica的仿真环境（如Dymola， OpenModelica）。
3.  **复现步骤**：
    *   在仿真环境中加载 `./model/BouncingBall.mo` 文件。
    *   创建一个新的仿真实验，设置停止时间（Stop Time）为10秒。
    *   选择Dassl求解器，并设置容差为1e-6。
    *   运行仿真。即可获得与本文档第7节一致的结果。
4.  **参数修改**：如需修改初始高度、恢复系数等参数，可以在模型的参数对话框或初始化脚本中进行相应设置后重新仿真。
