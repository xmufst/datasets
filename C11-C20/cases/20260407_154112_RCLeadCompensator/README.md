# RCLeadCompensator 模型文档

## 1. 模型目的与范围
本模型（`RCLeadCompensator`）模拟一个基于RC电路的电压跟踪闭环控制系统，并集成了超前补偿器（Lead Compensator）。模型的主要目的是展示如何利用Modelica标准库（MSL）中的组件，构建一个包含反馈控制、动态补偿和饱和保护的控制回路，以实现电容电压对给定参考电压信号的跟踪。

## 2. 组件清单（MSL 组件）
模型使用了以下来自 Modelica 标准库 (`Modelica.Blocks`) 的组件：
*   `Modelica.Blocks.Sources.Step`: 用于生成阶跃参考电压信号。
*   `Modelica.Blocks.Math.Feedback`: 用于计算参考信号与反馈信号之间的误差。
*   `Modelica.Blocks.Continuous.TransferFunction`: 实现超前补偿器的传递函数。
*   `Modelica.Blocks.Nonlinear.Limiter`: 对补偿器输出进行限幅，保护电路免受过压影响。
*   `Modelica.Blocks.Sources.RealExpression`: 将模型中的连续变量（电容电压 `Vc`）转换为信号，用于反馈连接。

## 3. 参数表
| 名称 | 值 | 单位 | 描述 |
| :--- | :--- | :--- | :--- |
| `R` | 10.0 | Ω | 电阻值 |
| `C_val` | 0.1 | F | 电容值 |
| `V_ref` | 5.0 | V | 参考电压（阶跃信号高度） |
| `Kc` | 5.0 | - | 补偿器增益（无量纲） |
| `tau_lead` | 0.5 | s | 超前时间常数 |
| `tau_lag` | 0.05 | s | 滞后时间常数 |
| `V_max` | 15.0 | V | 输入电压最大限幅值 |

## 4. 初始条件
| 变量 | 值 | 描述 |
| :--- | :--- | :--- |
| `Vc` | 0.0 V | 电容电压初始值 |

## 5. 控制逻辑说明
系统构成一个典型的单位负反馈闭环控制回路，具体逻辑如下：
1.  **参考生成**：一个阶跃信号源 (`step`) 在 `t=0` 时产生幅值为 `V_ref` 的参考电压。
2.  **误差计算**：反馈块 (`feedback`) 计算参考电压与实测电容电压 (`Vc`) 之间的误差 (`Vc_error`)。
3.  **动态补偿**：误差信号经过一个超前补偿器 (`compensator`)。该补偿器的传递函数为 `Kc * (tau_lead*s + 1) / (tau_lag*s + 1)`，用于改善系统的动态响应特性（如提高响应速度、改善稳定性）。
4.  **饱和限幅**：补偿器的原始输出 (`u_compensator`) 经过一个限幅器 (`limiter`)，其上下限为 `±V_max`，产生最终施加于RC电路的输入电压 `V_in`。
5.  **被控对象**：`V_in` 驱动一个简单的RC串联电路，其动态由方程 `C_val * der(Vc) = (V_in - Vc) / R` 描述。电容电压 `Vc` 通过 `RealExpression` 块被反馈回误差计算环节，形成闭环。

## 6. 仿真设置
基于运行元数据，本次仿真的具体设置如下：

## 7. 结果概览
下表列出了关键目标变量在整个仿真时间（0~10秒）内的统计摘要。

| 变量 | 单位 | 初始值 | 终值 | 最小值 | 最大值 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `Vc` | V | 0.0 | ~4.167 | 0.0 | ~4.167 | 电容电压，系统输出 |
| `Vc_error` | V | 5.0 | ~0.833 | ~0.833 | 5.0 | 跟踪误差 (`V_ref - Vc`) |
| `V_in` | V | 15.0 | ~4.167 | ~4.167 | 15.0 | 施加于RC电路的实际电压（限幅后） |
| `i_R` | A | 1.5 | ~0.0 | ~-5.73e-7 | 1.5 | 流经电阻的电流 |
| `u_compensator` | - | 250.0 | ~4.167 | ~4.167 | 250.0 | 补偿器原始输出（限幅前） |

**结果分析**: 系统在阶跃参考 `V_ref=5V` 作用下，输出 `Vc` 最终稳定在约 `4.167V`，存在稳态误差。这是由于一阶被控对象在比例控制下的固有特性。仿真初期，误差较大导致补偿器输出饱和 (`u_compensator=250`)，限幅器输出 `V_in` 达到上限 `15V`，使系统以最大能力响应。

## 8. 运行与复现方法
1.  **打开模型**：在您的 Modelica 仿真环境（如 Dymola, OpenModelica）中，打开模型文件 `./model/RCLeadCompensator.mo`。
2.  **检查参数**：确认模型参数与第3节中的参数表一致。如有需要，可在仿真前进行修改。
3.  **设置仿真**：在仿真设置中，配置以下参数：
    *   停止时间：`10.0`
    *   数值积分方法：`dassl`
    *   容差：`1e-6`
    *   输出间隔（可选）：`0.01`
4.  **运行仿真**：执行仿真。
5.  **查看结果**：仿真结束后，绘制变量 `Vc`, `V_ref`, `V_in`, `Vc_error` 等以观察系统动态响应。
6.  **评审说明**：根据评审报告，模型通过了方程和结果检查。参数和逻辑检查存在警告（非错误），建议在应用前结合控制理论审查补偿器参数 (`Kc`, `tau_lead`, `tau_lag`) 设计的合理性。
