# FurnaceFeedforward 模型文档

## 1. 模型目的与范围
本模型 `FurnaceFeedforward` 旨在模拟一个采用纯前馈（Feedforward）扰动补偿控制的炉子系统。其核心目的是研究如何在热扰动（例如，一个阶跃形式的热负载）作用于系统时，通过前馈控制路径实时调整加热功率，以理想地维持炉内温度恒定。模型的范围涵盖了集总参数热力学模型、基本控制逻辑（前馈补偿）的搭建以及系统对阶跃扰动的动态响应分析。

涉及的物理机制包括：
*   集总热容模型
*   热平衡方程
*   牛顿冷却定律（热损失）

## 2. 组件清单（MSL 组件）
本模型使用了以下来自 Modelica 标准库（MSL）的组件：
*   `Modelica.Blocks.Sources.Constant`: 产生恒定的稳态加热功率设定值 (`Q_ss`)。
*   `Modelica.Blocks.Sources.Step`: 在指定时间 (`t_dist`) 产生一个阶跃形式的扰动功率 (`Q_dist`)。
*   `Modelica.Blocks.Math.Gain`: 作为前馈增益 (`K_ff`)，按比例调整扰动信号以生成补偿量。
*   `Modelica.Blocks.Math.Add`: 将稳态加热功率与前馈补偿信号相加，得到最终作用于炉子的总加热功率 (`Q_heater`)。

## 3. 参数表
下表列出了模型的所有可调参数。

| 参数名 | 值 | 单位 | 描述 |
| :--- | :--- | :--- | :--- |
| `C_furnace` | 5000.0 | J/K | 炉子热容 |
| `K_heater` | 100.0 | W/unit | 加热器功率增益 (W/控制量单位) |
| `UA` | 5.0 | W/K | 热损失系数 (传热系数 × 面积) |
| `T_amb` | 20.0 | °C | 环境温度 |
| `Q_ss` | 300.0 | W | 稳态基础加热功率 |
| `T0` | 80.0 | °C | 初始温度参数 |
| `Q_dist` | 200.0 | W | 扰动功率的阶跃幅值 |
| `t_dist` | 100.0 | s | 扰动施加的时间点 |
| `K_ff` | 1.0 | 1 (无量纲) | 前馈增益 |

**⚠️ 注意**：审查报告指出存在一个参数相关的问题。需求说明中定义了 `K_heater` 参数，但在当前模型代码的实现中，总加热功率 `Q_heater` 直接由控制信号（单位：W）赋值，并未使用 `K_heater` 参数进行单位转换。这与需求说明存在不一致，建议在模型方程或控制信号路径中引入此增益以符合原始设计意图。

## 5. 控制逻辑说明
控制系统采用纯前馈结构，旨在对可测量的扰动进行补偿。
1.  **基准信号**：一个恒值模块提供稳态所需的加热功率 `Q_ss`。
2.  **扰动检测**：一个阶跃模块模拟在 `t=100 s` 时发生的、幅值为 `200 W` 的热扰动 (`Q_disturbance`)。
3.  **前馈补偿**：检测到的扰动信号经过一个增益为 `K_ff` 的前馈控制器。在本案例中 `K_ff = 1`，意味着控制器旨在产生一个与扰动幅值相等但方向相反的补偿功率 (`u_ff`)。
4.  **功率合成**：稳态功率与前馈补偿功率在加法器中相加，其输出 (`Q_heater`) 直接作为总加热功率输入到炉子物理模型中。
5.  **控制目标**：理想情况下，前馈补偿量 `u_ff` 完全抵消扰动 `Q_disturbance`，使得炉子的净热输入 (`Q_heater - Q_disturbance`) 在扰动发生后保持不变，从而维持温度 `T` 恒定。

## 6. 仿真设置
本次仿真运行采用了以下设置：
*   **停止时间**: 300 s
*   **输出/通信间隔**: 0.01 s
*   **求解器**: DASSL
*   **相对容差**: 1e-6

## 7. 结果概览
以下为目标变量在 `0~300 s` 仿真周期内的统计摘要。结果验证了在前馈增益 `K_ff=1` 的理想情况下，系统实现了对阶跃扰动的完美补偿。

| 变量 | 单位 | 起始值 | 最终值 | 最小值 | 最大值 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **`T` (炉温)** | °C | 80.0 | **80.0** | 80.0 | 80.0 |
| **`Q_heater` (总加热功率)** | W | 300.0 | **500.0** | 300.0 | 500.0 |
| **`Q_loss` (热损失功率)** | W | 300.0 | **300.0** | 300.0 | 300.0 |
| **`Q_disturbance` (扰动功率)** | W | 0.0 | **200.0** | 0.0 | 200.0 |
| **`u_ff` (前馈补偿信号)** | W | 0.0 | **200.0** | 0.0 | 200.0 |

**结果分析**：
炉温 `T` 在整个仿真过程中保持恒定的 80°C。在 `t=100 s` 时，`Q_disturbance` 从 0 W 阶跃至 200 W。与此同时，前馈控制器立即生成了 `u_ff = 200 W` 的补偿信号，使得总加热功率 `Q_heater` 从 300 W 同步增加至 500 W。净加热功率 (`Q_heater - Q_disturbance`) 始终保持为 300 W，恰好等于稳态热损失 `Q_loss`，因此系统能量平衡，温度无波动。这证明了在当前参数下前馈控制达到了理想效果。

## 8. 运行与复现方法
1.  **准备模型**：将提供的 `FurnaceFeedforward.mo` 文件置于您的 Modelica 项目目录下（例如 `./model/` 文件夹）。
2.  **加载模型**：在您的 Modelica 仿真环境（如 Dymola, OpenModelica）中，加载或打开模型文件 `./model/FurnaceFeedforward.mo`。
3.  **检查参数（可选）**：您可以根据第3节的参数表修改模型参数以研究不同工况。**请注意**，模型代码中 `K_heater` 参数未被使用，若需使其生效，需按第3节注意中的建议修改模型。
4.  **设置仿真**：在实验注解中已预设仿真参数（停止时间300s，容差1e-6等）。在仿真环境中通常可直接使用这些设置。
5.  **执行仿真**：编译并仿真模型 `FurnaceFeedforward`。
6.  **查看结果**：仿真完成后，绘制第7节中列出的目标变量（`T`, `Q_heater`, `Q_disturbance`, `u_ff`），以复现文档中描述的结果。

使用此步骤，您应能获得与本文档“结果概览”一节中一致的仿真数据。
