# DCMotorStateFeedback 模型文档

## 1. 模型目的与范围

本模型 `DCMotorStateFeedback` 实现了一个直流电机（DC Motor）状态反馈速度跟踪控制系统。模型包含完整的电机物理（电磁、机械）模块以及基于状态反馈的控制回路，用于研究直流电机在参考速度阶跃输入和负载转矩扰动下的动态响应与控制性能。

主要研究范围包括：
*   直流电机的物理建模（电枢回路、反电动势、电磁转矩、转动惯量、粘滞摩擦）。
*   全状态反馈控制律的设计与实现，用于速度跟踪。
*   系统对参考输入和外部负载扰动的动态响应分析。

## 2. 组件清单 (Modelica Standard Library)

模型使用了以下来自 Modelica 标准库（MSL）的组件：

*   **Modelica.Blocks.Sources.Step** (`referenceStep`, `loadTorqueStep`): 用于生成参考速度信号和负载转矩扰动信号。
*   **Modelica.Blocks.Math.Gain** (`feedforwardGain`, `currentFeedbackGain`, `speedFeedbackGain`): 分别实现参考前馈增益、电流反馈增益和速度反馈增益。
*   **Modelica.Blocks.Math.Add3** (`controlVoltageAdd`): 三输入加法器，用于合成控制电压。
*   **Modelica.Blocks.Sources.RealExpression** (`currentSignal`, `speedSignal`): 将模型内部变量 `ia` 和 `omega` 转换为信号块，以便接入控制系统。

## 4. 初始条件

模型变量的初始条件设置如下：

*   **`ia` (电枢电流)**: `0 A`
*   **`omega` (转子速度)**: `0 rad/s`

这些初始条件在模型变量的声明中通过 `start` 属性定义。

## 5. 控制逻辑说明

控制器采用全状态反馈控制律，旨在使电机速度 `omega` 跟踪参考值 `omega_ref`。

1.  **反馈通道**:
    *   测量电枢电流 `ia` 和转子速度 `omega`。
    *   分别乘以对应的反馈增益 `K1` 和 `K2`，得到反馈信号。

2.  **前馈通道**:
    *   将参考速度 `omega_ref` 乘以前馈增益 `Kr`，得到前馈信号。

3.  **控制信号合成**:
    *   控制电压 `Va` 由前馈信号减去两个反馈信号得到，其控制律为：
        `Va = Kr * omega_ref - K1 * ia - K2 * omega`
    *   该控制律由一个 `Add3` 模块实现。

4.  **抗扰动**:
    *   控制器通过电流和速度的反馈作用，自动补偿负载转矩 `T_load` 的扰动。

## 6. 仿真设置

本次仿真采用的设置如下：

*   **仿真时长**: `0.0` 至 `10.0` 秒。
*   **通信间隔 (步长)**: `0.01` 秒。
*   **求解器**: `dassl` (DAE 求解器)。
*   **容差**: `1e-06`。
*   **仿真脚本**: 使用 `run_single.py` 脚本执行。

## 7. 结果概览

下表列出了关键目标变量在整个仿真过程中 (`0-10 s`) 的统计摘要：

| 变量 | 单位 | 初始值 | 最终值 (t=10s) | 最小值 | 最大值 | 备注 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `omega` | rad/s | 0.00 | **99.59** | 0.00 | 100.81 | 速度最终接近参考值100 rad/s，存在微小稳态误差。 |
| `ia` | A | 0.00 | **1.50** | 0.00 | 38.11 | 启动瞬间电流冲击大，稳态电流用于平衡负载转矩。 |
| `Va` (或 `u_control`) | V | 62.00 | **11.46** | 11.09 | 62.00 | 启动时提供高电压，稳态时用于维持电流和克服反电动势。 |
| `Vemf` | V | 0.00 | **9.96** | 0.00 | 10.08 | 与速度成正比，变化趋势与 `omega` 一致。 |
| `T_motor` | N·m | 0.00 | **0.15** | 0.00 | 3.81 | 与电流 `ia` 成正比，最大值对应电流峰值。 |
| `T_load` | N·m | 0.00 | **0.05** | 0.00 | 0.05 | 在 t=5s 时发生阶跃。 |
| `omega_error` | rad/s | 100.00 | **0.41** | -0.81 | 100.00 | 速度跟踪误差，最终趋近于一个小的正值，表明存在轻微欠调。 |

**结果分析要点**：
*   **启动过程 (0s附近)**: 系统响应迅速，速度超调约0.8%，随后快速收敛。
*   **稳态精度 (5s前)**: 在负载施加前，速度 `omega` 最终稳定在约99.6 rad/s，稳态误差约0.4 rad/s。
*   **抗扰过程 (5s)**: 在 t=5s 施加负载转矩后，速度瞬间下降，控制器通过增大电枢电流 `ia` 和控制电压 `Va` 进行补偿，使速度恢复至接近原稳态值，体现了控制系统的抗干扰能力。

## 8. 运行与复现方法

### 模型文件
*   主模型文件: `./model/DCMotorStateFeedback.mo`

### 运行仿真 (以 Dymola 为例)
1.  打开 Dymola。
2.  加载模型文件 `./model/DCMotorStateFeedback.mo`。
3.  在 `Simulation Setup` 中设置：
    *   Stop time: `10.0`
    *   Output Interval: `0.01`
    *   Method: `Dassl`
    *   Tolerance: `1e-06`
4.  点击 `Simulate` 进行仿真。

### 使用脚本复现
项目提供了自动化脚本 `run_single.py`。确保安装 Dymola 和 DymolaPython 接口后，在项目根目录下运行以下命令即可复现仿真：

```bash
python run_single.py
```

该脚本会自动调用 Dymola，加载模型，应用本报告中的仿真设置，执行仿真，并保存结果。
